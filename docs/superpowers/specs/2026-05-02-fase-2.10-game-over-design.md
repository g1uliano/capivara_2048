## Feature A — Animação Piscante + Haptic na `GameOverItemOverlay`

### Objetivo

Chamar atenção do jogador para o item disponível com pulso visual + tátil.

### Arquivos modificados

- `lib/presentation/screens/game/game_over_item_overlay.dart`

### Implementação

Converter o widget para `ConsumerStatefulWidget`.

**Animação visual — `flutter_animate`:**

```dart
Image.asset(itemPngPath, width: 120, height: 120)
  .animate(
    key: ValueKey(currentItemIndex), // reset automático ao trocar item
    onPlay: (c) => c.repeat(reverse: true),
  )
  .fadeIn(duration: 400.ms, curve: Curves.easeInOut)
  .then()
  .fadeOut(duration: 400.ms, curve: Curves.easeInOut)
```

Ciclo: opacidade 1.0 → 0.4 → 1.0, 800ms, `easeInOut`. Trocar `currentItemIndex` via `setState` muda a `ValueKey` e reinicia automaticamente — sem `AnimationController` manual para o visual.

**Haptic — `AnimationController` separado:**

```dart
late final AnimationController _hapticController;

@override
void initState() {
  super.initState();
  _hapticController = AnimationController(
    vsync: this,
    duration: const Duration(milliseconds: 800),
  )
    ..addStatusListener((status) {
      if (status == AnimationStatus.completed) {
        maybeHaptic(ref); // respeita settings.hapticEnabled
      }
    })
    ..repeat();
}

@override
void dispose() {
  _hapticController.dispose();
  super.dispose();
}
```

**Troca de item ("Próximo item"):**

```dart
setState(() => currentItemIndex++);
_hapticController.reset();
_hapticController.repeat();
```

**Ao tocar "Usar item":**

```dart
_hapticController.stop();
// ... lógica existente de uso do item
```

**Haptic de entrada (Game Over):** o `repeat()` no `initState` dispara o primeiro `completed` após 800ms — equivale ao haptic sincronizado com o primeiro pulso. Não há haptic extra de "entrada" separado; o pulso é o sinal.

### Regras

- Animação e haptic param imediatamente ao tocar "Usar item"
- Troca de item reinicia ambos
- `dispose()` obrigatório — sem vazamento de controller
- Haptic respeita `settings.hapticEnabled` via `maybeHaptic(ref)`

### Casos de teste

```dart
testWidgets('ícone pisca em loop enquanto item exibido');
testWidgets('ao tocar Usar item, animação para antes da navegação');
testWidgets('ao trocar item, ValueKey muda e animação reinicia');
testWidgets('dispose não lança exceção');
testWidgets('haptic não dispara se hapticEnabled == false');
testWidgets('haptic dispara a cada ciclo se hapticEnabled == true');
```

## Feature B — `GameOverNoItemsOverlay`

### Objetivo

Quando o tabuleiro trava e o inventário está vazio, oferecer ao jogador três caminhos antes de confirmar o Game Over: ver anúncio, comprar item avulso, ou encerrar.

### Arquivos

- **Novo:** `lib/presentation/widgets/game_over_no_items_overlay.dart`
- **Modificado:** `lib/presentation/screens/game/game_screen.dart` — wiring
- **Modificado:** `lib/data/shop_data.dart` — adicionar `kItemUnitPrices`

### Estado local

```dart
late final ItemType _drawnItem;
late final String _itemName;
late final String _itemDescription;
late final String _itemPngPath;
late final double _itemPrice;

@override
void initState() {
  super.initState();
  _drawnItem = const [
    ItemType.bomb2,
    ItemType.bomb3,
    ItemType.undo1,
    ItemType.undo3,
  ][Random().nextInt(4)];
  // derivar name, description, pngPath, price a partir de _drawnItem
}
```

Item sorteado uma única vez em `initState`, imutável durante vida do widget.

### Fluxo — Opção 1: Ver anúncio

1. Toca "📺 Ver anúncio e receber [NomeItem]"
2. Se `adWatchedToday >= 40`: botão desabilitado com label "Limite diário atingido" — fim
3. `await FakeAdService.showRewardedAd()` (dev) / `google_mobile_ads` (produção)
4. Se retornar `true` (rewarded callback):
   - `inventoryNotifier.add(_drawnItem, 1)`
   - `livesNotifier.incrementAdWatched()`
   - Overlay fecha
   - `SnackBar("[NomeItem] adicionado! Boa sorte! 🎉")`
   - Partida continua — **nenhuma vida consumida**
5. Se retornar `false` (skip/falha): overlay permanece, sem entrega

### Fluxo — Opção 2: Comprar

1. Toca "🛒 Comprar [NomeItem]  •  R$ X,XX"
2. `AlertDialog`:
   - Título: `"Confirmar compra"`
   - Corpo: `"Você receberá 1× [NomeItem] por R$ X,XX"`
   - Botões: "Cancelar" / "Confirmar compra"
3. Cancelar → fecha dialog, retorna ao overlay
4. Confirmar (mock dev):
   - `inventoryNotifier.add(_drawnItem, 1)`
   - Overlay fecha
   - `SnackBar("[NomeItem] adicionado! Boa sorte! 🎉")`
   - Partida continua — **nenhuma vida consumida**
5. Confirmar (produção — Fase 3): SKU individual via `in_app_purchase`; em sucesso, entrega 1× item

### Fluxo — Opção 3: Encerrar

1. Toca "Encerrar partida"
2. `AlertDialog`:
   - Corpo: `"Tem certeza? Você perderá 1 vida e a partida será encerrada."`
   - Botões: "Cancelar" / "Confirmar"
3. Cancelar → retorna ao overlay
4. Confirmar:
   - `livesNotifier.consume(1)`
   - `gameNotifier.setAwaitingResolution(false)`
   - Overlay fecha → modal padrão de Game Over

### Wiring em `game_screen.dart`

```dart
// Fase 2.9 — inalterado:

  const Positioned.fill(child: GameOverItemOverlay()),

// Fase 2.10 — novo:

// Modal existente — inalterado:

  ...
```

Os dois fluxos são mutuamente exclusivos: `GameOverItemOverlay` quando `hasAnyItem == true`, `GameOverNoItemsOverlay` quando `hasAnyItem == false`.

| # | Critério |
|---|---|
| A1 | Ícone do item pisca em loop durante toda a exibição na `GameOverItemOverlay` |
| A2 | Animação para ao tocar "Usar item" |
| A3 | Troca de item reinicia a animação piscante no novo item |
| A4 | Haptic dispara a cada ciclo quando `hapticEnabled == true` |
| A5 | Haptic não dispara quando `hapticEnabled == false` |
| A6 | Nenhum vazamento de `AnimationController` |
| B1 | Overlay aparece somente quando `inventory.isEmpty == true` no game over |
| B2 | Item sorteado é um dos 4 tipos; não muda enquanto overlay está aberto |
| B3 | Opção de anúncio desabilitada quando `adWatchedToday >= 40` |
| B4 | Anúncio completo entrega o item sorteado e fecha o overlay sem consumir vida |

| B6 | Compra (mock) entrega o item sorteado e fecha o overlay sem consumir vida |

| Dispositivo | Resolução | Checklist |
|---|---|---|
| Android pequeno | 360×640 | overflow em cards? botões cortados? |
| iPhone 14 | 390×844 | SafeArea correta? bounce do ícone suave? |

**Fluxos a testar manualmente:**
1. Game over com itens → `GameOverItemOverlay` pisca → usar item → partida continua
2. Game over com itens → trocar item → animação reinicia
3. Game over com itens → desistir → modal Game Over → 1 vida consumida
4. Game over sem itens → `GameOverNoItemsOverlay` → ver anúncio → item entregue → continua
