# Spec — Fase 2.3.11: Anfitrião inicial Tanajura + Fundo unificado

**Data:** 2026-04-28
**Versão alvo:** v0.8.0
**Fase anterior concluída:** 2.3.10 (v0.7.0) — 152 testes passando

---

## Contexto

Duas correções de polimento visual:

1. O `HostBanner` exibe a Tanajura desde o boot em vez do placeholder "Comece!".
2. A `HomeScreen` usa o mesmo `fundo.png` da `GameScreen` para continuidade visual.

### Nota sobre nomenclatura

O campo no código é `maxLevel` (em `GameState`), não `highestLevelReached` como referenciado na spec geral. Todos os itens abaixo usam o nome do código.

### Nota sobre persistência

`GameState` não tem persistência (sem Hive, sem save file). Não há migration necessária para esta fase.

---

## Sub-entrega A+B — `maxLevel = 1` + remover `_Placeholder`

### Decisões

- `maxLevel` inicia em `1` tanto em `newGame()` quanto em `restart()`.
- `_Placeholder` é removido. `HostBanner` sempre renderiza `_AnimalHost`.
- `AnimatedSwitcher` permanece para animar trocas futuras de anfitrião (nível 2+).
- Tanajura aparece **estática** no boot — sem animação fade+scale, pois não há troca.
- `animalForLevel(0)` lança `ArgumentError`; com `maxLevel >= 1` garantido, nunca é chamado.

### Arquivos a modificar

| Arquivo | Mudança |
|---|---|
| `lib/domain/game_engine/game_engine.dart` | `GameState(... maxLevel: 1 ...)` em `newGame()` |
| `lib/presentation/controllers/game_notifier.dart` | `maxLevel: 1` em `restart()` |
| `lib/presentation/widgets/host_banner.dart` | Remover `_Placeholder`, remover branch `maxLevel == 0`, sempre renderizar `_AnimalHost` |
| `lib/presentation/screens/game/game_screen.dart` | Remover guard `maxLevel > 0`, chamar `animalForLevel(state.maxLevel)` diretamente |

### Testes obrigatórios

- `game_engine_test.dart`: `newGame()` → `state.maxLevel == 1`
- `game_notifier_test.dart`: `restart()` → `state.maxLevel == 1`
- `host_banner_test.dart`: substituir `'placeholder mostra texto Comece! quando maxLevel == 0'` por `'exibe Tanajura no estado inicial (maxLevel == 1)'`
- Auditar todos os testes que asseram `maxLevel == 0` após `newGame()` ou `restart()` e corrigir para `== 1`

### Critérios de aceite

- App sobe → HostBanner exibe "Tanajura" imediatamente, sem placeholder.
- `restart()` → HostBanner volta a exibir "Tanajura".
- Nenhuma `ArgumentError` em nenhum caminho de código.
- Todos os testes passam.

## Sub-entrega D — Fundo unificado

### Decisões

- Remover parâmetro `animal` de `GameBackground` (dead code — nunca foi usado internamente).
- `GameBackground` vira `const GameBackground({required Widget child})`.
- `HomeScreen` envolve seu corpo com `GameBackground`; `Scaffold` interno usa `backgroundColor: Colors.transparent`.
- Sem overlay na HomeScreen — fundo idêntico à `GameScreen`. Contraste dos elementos da Home auditado na implementação; ajustado caso a caso (OutlinedText, sombras) se necessário.
- `fundo.png` já está em cache no boot (`app.dart` → `didChangeDependencies`). Sem flash na HomeScreen.

### Arquivos a modificar

| Arquivo | Mudança |
|---|---|
| `lib/presentation/widgets/game_background.dart` | Remover parâmetro `animal`; tornar widget completamente `const` sem deps externas |
| `lib/presentation/screens/game/game_screen.dart` | Atualizar call site: `GameBackground(child: ...)` sem `animal` |
| `lib/presentation/screens/home_screen.dart` | Substituir `Scaffold(backgroundColor: AppColors.primary)` por `GameBackground` envolvendo o corpo |

### Testes obrigatórios

- `game_background_test.dart`: atualizar call sites que passavam `animal` para a nova assinatura sem parâmetro.
- Snapshots da `HomeScreen`: regenerar golden files após mudança de background.
- Smoke test visual: HomeScreen renderiza sem flash (fundo aparece imediatamente).

### Critérios de aceite

- `HomeScreen` exibe `fundo.png` idêntico ao da `GameScreen`.
- Sem parâmetro `animal` exposto em `GameBackground`.
- Todos os elementos da Home legíveis sobre o novo fundo (sem texto ilegível, botões visíveis).
- Todos os testes passam (incluindo goldens regenerados).
