## Fix C — Overflow na GameScreen

**Problema:** `BoardWidget` calcula `boardSize = screenWidth - 24`. Em telas com pouca altura disponível (ex: 360×640), o board de 336×336 + `GameHeader` + `InventoryBar` ultrapassa a altura do `SafeArea`, cortando a 4ª linha do tabuleiro.

**Arquivos a modificar:**
- `lib/presentation/widgets/board_widget.dart`
- `lib/presentation/screens/game/game_screen.dart`

**Mudanças:**

`board_widget.dart`:
- Adicionar parâmetro opcional `double? size` ao construtor
- Substituir `final boardSize = screenWidth - GameConstants.boardPadding * 2` por `final boardSize = size ?? (screenWidth - GameConstants.boardPadding * 2)`

`game_screen.dart`:
- Envolver o `GestureDetector` (que contém `BoardWidget`) em `LayoutBuilder`
- Calcular `final boardSide = min(constraints.maxWidth - 24, constraints.maxHeight - inventoryBarEstimatedHeight)` — usar `dart:math` `min`
- Passar `size: boardSide` ao `BoardWidget`

**Critério de aceite:**
- Tabuleiro 4×4 completo visível em dispositivo 360×640dp
- Sem regressão em 412×892dp
- Sem `RenderFlex overflowed` no widget tree

**Testes:**
- `test/presentation/game_screen_layout_test.dart` — novo `testWidgets` com `tester.view.physicalSize = Size(360, 640)` verificando ausência de overflow e presença de 4 linhas no board

## Fix A — Badge inconsistente no card de Recompensa Diária

**Problema:** quando `showBadge: true`, `_HomeCard` retorna `Stack` em vez do `Container` diretamente, causando tamanho diferente dos outros cards no `GridView` e quebrando o alinhamento.

**Arquivo a modificar:**
- `lib/presentation/screens/home_screen.dart` — classe `_HomeCard`

**Mudanças:**
- Envolver o `card` (após o bloco `if (comingSoon)`, antes do `if (showBadge)`) em `SizedBox.expand()` para garantir que o widget base preenche 100% do espaço do grid
- O `Stack` usa `Positioned.fill` no filho principal (o `card`)
- Badge: `Positioned(top: -6, right: -6)` com `Container(width: 20, height: 20, decoration: BoxDecoration(color: Colors.red, shape: BoxShape.circle), child: Center(child: Text('!', style: TextStyle(color: Colors.white, fontSize: 12, fontWeight: FontWeight.bold))))`
- `Stack(clipBehavior: Clip.none, ...)` — já presente, manter

**Critério de aceite:**
- Grid da Home alinhado uniformemente com e sem badge visível
- Badge "!" vermelho visível no canto superior direito do card quando recompensa disponível
- Badge ausente quando recompensa não disponível

**Testes:**
- `test/presentation/home_screen_test.dart` — novo teste verificando presença do texto `'!'` quando badge ativo
- `test/presentation/home_screen_test.dart` — novo teste verificando que `'Recompensa Diária'` continua presente com badge ativo

## Fix B — Textos ilegíveis sobre fundo dinâmico

**Problema:** dois textos com cor `AppColors.primary` renderizados diretamente sobre `GameBackground` sem container opaco:
1. `CollectionScreen`: `'$highest/11 animais descobertos'`
2. `SettingsScreen`: títulos de seção via `_SettingsSection` ("Geral", "Áudio", "Sobre")

**Arquivos a modificar:**
- `lib/presentation/screens/collection_screen.dart`
- `lib/presentation/screens/settings_screen.dart`

**Mudanças:**

`collection_screen.dart`:
- Substituir `Text('$highest/11 animais descobertos', style: GoogleFonts.fredoka(fontSize: 16, color: AppColors.primary))` por `OutlinedText('$highest/11 animais descobertos', style: GoogleFonts.fredoka(fontSize: 16))`
- Adicionar import de `outlined_text.dart`

`settings_screen.dart` — classe `_SettingsSection`:
- Substituir `Text(title, style: GoogleFonts.fredoka(fontSize: 14, color: AppColors.primary, fontWeight: FontWeight.w600))` por `OutlinedText(title, style: GoogleFonts.fredoka(fontSize: 14, fontWeight: FontWeight.w600))`
- Adicionar import de `outlined_text.dart`

**Nota:** `OutlinedText` sobrescreve `color` para branco internamente via `outlinedWhiteTextStyle`. Remover `color: AppColors.primary` do style passado.

**Critério de aceite:**
- Contador de animais e títulos de seção legíveis sobre qualquer variação do fundo dinâmico
- Labels dos cards da Home (dentro de container branco) — não alterar

**Testes:**
- `test/presentation/collection_screen_test.dart` — verificar `find.byType(OutlinedText)` presente para o contador
- `test/presentation/settings_screen_test.dart` — verificar `find.byType(OutlinedText)` para título `'Geral'`

## Fix D — Controles de Configurações ilegíveis

**Problema:** `SettingsScreen` usa `ListView` com controles renderizados diretamente sobre `GameBackground`. Texto e widgets interativos (`Switch`, `SegmentedButton`, `Slider`) sem fundo opaco ficam ilegíveis.

**Arquivo a modificar:**
- `lib/presentation/screens/settings_screen.dart`

**Mudanças:**
- Remover `Divider` entre seções (cards já criam separação visual)
- Agrupar controles de cada seção em `Card(color: Colors.white.withOpacity(0.88), shape: RoundedRectangleBorder(borderRadius: BorderRadius.circular(12)), elevation: 2)`
- Dentro de cada card: `SwitchListTile` e `ListTile` com `tileColor: Colors.transparent`
- Estrutura:
  - Card "Geral": `SwitchListTile` (Vibração) + `ListTile` (Idioma + `SegmentedButton`)
  - Card "Áudio": `ListTile` label "Disponível na Fase 5" + dois `ListTile` com sliders em `Opacity(0.4)`
  - Card "Sobre": `ListTile` versão + `ListTile` copyright
- Títulos de seção (`_SettingsSection`) ficam fora dos cards — já cobertos pelo Fix B
- `margin` dos cards: `EdgeInsets.symmetric(horizontal: 12, vertical: 4)`

**Critério de aceite:**
- Todos os controles legíveis sobre fundo dinâmico
- Switch e SegmentedButton funcionais
- Sliders visíveis mas desabilitados com label "Disponível na Fase 5"
- Sem overflow na tela

**Testes:**
- `test/presentation/settings_screen_test.dart` — verificar presença de pelo menos um `Card` no widget tree
- `test/presentation/settings_screen_test.dart` — verificar `OutlinedText` com `'Geral'` presente
