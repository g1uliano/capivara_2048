# Fase 2.12 — Expansão além do 2048: Peixe-boi, Jacaré e Ranking Lendas

**Data:** 2026-05-03  
**Status:** Aprovado — pronto para implementação  
**Versão alvo:** v1.2.0

## Objetivo

Expandir o jogo além do marco 2048 com dois novos animais (Peixe-boi/4096 e Jacaré/8192), um fluxo de escolha ao atingir cada marco, anfitrião dinâmico, sistema de `PersonalRecords` persistido, ranking local de partidas e ranking mock de Lendas — com interface pronta para substituição por Firebase na Fase 3.

---

## Decisões de design

| Ponto | Decisão |
|---|---|
| Modelagem vitória múltipla | `pendingMilestone: int?` no `GameState`; `hasWon = true` apenas ao encerrar |
| Timer após Continuar | Continua rodando; captura `bestTimeMs2048` ao atingir nível 11 e `bestTimeMs4096` ao atingir nível 12 |
| Recompensa 4096 (1ª vez) | Entregue imediatamente ao exibir o `VictoryChoiceDialog` |
| Recompensa 8192 (1ª vez) | Entregue ao encerrar (`endGame()`) |
| Ranking Lendas | Puramente prestígio — sem reset semanal, sem recompensa periódica |
| Ranking local — campos | Por tempo = menor `elapsedMs`; por pontuação = maior `score` |
| Injeção do ranking | `kDebugMode` → `FakeRankingService`; Fase 3 substitui por `FirebaseRankingService` |

## Sub-B — Fluxo de vitória múltipla

### `GameState`

Três novos campos:

```dart
final int? pendingMilestone;   // 11, 12 ou 13 — dispara VictoryChoiceDialog
final int? bestTimeMs2048;     // elapsedMs capturado ao atingir nível 11
final int? bestTimeMs4096;     // elapsedMs capturado ao atingir nível 12
```

Incluídos em `copyWith`. Excluídos de `==`/`hashCode` se o padrão do projeto for esse (seguir convenção existente).

### `game_engine.dart`

`_checkWin` passa a retornar sempre `false` — a lógica de vitória sobe para o `GameNotifier`. O engine continua gerando tiles de nível > 11 normalmente (sem cap).

### `game_notifier.dart`

Na transição de estado pós-move, detectar novos marcos:

```
para level in [11, 12, 13]:
  se tile.level == level apareceu pela 1ª vez nesta partida
  && pendingMilestone == null
  && hasWon == false:
    setar pendingMilestone = level
    se level == 11: capturar bestTimeMs2048 = elapsedMs
    se level == 12: capturar bestTimeMs4096 = elapsedMs
    chamar personalRecordsNotifier.recordMilestone(level, DateTime.now())
```

Dois novos métodos no `GameNotifier`:

**`dismissMilestone()`** — chamado ao "Continuar":
```
state = state.copyWith(pendingMilestone: null)
```
Timer não é tocado (continua rodando).

**`endGame()`** — chamado ao "Encerrar":
```
pausar timer
state = state.copyWith(hasWon: true, pendingMilestone: null)
gravar GameRecord em Hive
```

**Guard de pause:**
```dart
void pause() {
  if (state.isGameOver || state.hasWon || state.pendingMilestone != null) return;
  ...
}
```

Guard em `onTapWhenEmpty` da `GameScreen`: não abre `ShopOverlay` se `pendingMilestone != null`.

### `VictoryChoiceDialog`

**Arquivo novo:** `lib/presentation/widgets/victory_choice_dialog.dart`

```dart
class VictoryChoiceDialog extends ConsumerWidget {
  final int milestone; // 11, 12 ou 13
}
```

Layout: `PopScope(canPop: false)` envolvendo um `Dialog` centralizado com título, subtítulo e botões.

| Marco | Título | Subtítulo | Botões |
|---|---|---|---|
| 11 | "Capivara Lendária! 🎉" | "Você chegou ao 2048!" | Continuar / Encerrar |
| 12 | "Peixe-boi! Incrível! 🌊" | "Você chegou ao 4096!" | Continuar / Encerrar |
| 13 | "Jacaré! Lendário! 🐊" | "Você chegou ao 8192!" | Encerrar |

- Botão voltar Android bloqueado (`PopScope(canPop: false)`)
- Timer **não é pausado** durante o diálogo
- "Continuar" → `notifier.dismissMilestone()`; se marco == 12 (4096) e `isFirstTime(12)` → entregar recompensa antes de dispensar
- "Encerrar" → `notifier.endGame()`; se marco == 13 (8192) e `isFirstTime(13)` → entregar recompensa dentro de `endGame()`

### Recompensas de primeira vez

| Marco | Itens |
|---|---|
| 4096 | 5 vidas + 3 bombas + 3 desfazer |
| 8192 | 10 vidas + 5 bombas + 5 desfazer |

Entrega via providers existentes (`livesProvider`, `inventoryProvider`). Flag `rewardCollected4096/8192` em `PersonalRecords` garante idempotência.

### Render na `GameScreen`

```dart
if (state.pendingMilestone != null && !state.hasWon)
  Positioned.fill(
    child: VictoryChoiceDialog(milestone: state.pendingMilestone!),
  ),
```

Posicionado após o `GameOverModal` no Stack (mas `pendingMilestone` e `hasWon` são mutuamente exclusivos por design).

## Plano de validação em dispositivo

- **360×640** (Android compacto): `VictoryChoiceDialog` legível sem overflow; aba Lendas sem scroll horizontal
- **390×844** (iPhone padrão): `RankingScreen` com tabs secundários sem corte; `SafeArea` correta
- **Fluxo completo:** partida → 2048 (Continuar) → 4096 (Continuar) → 8192 (Encerrar) → recompensas entregues → `PersonalRecords` atualizado → ranking pessoal com entrada → anfitrião Jacaré
