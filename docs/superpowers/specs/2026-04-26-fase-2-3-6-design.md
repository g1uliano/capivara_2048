**Arquivos novos:** `lib/core/providers/reduce_effects_provider.dart`  
**Arquivos modificados:** `lib/presentation/widgets/pause_overlay.dart`, `lib/presentation/screens/game/game_screen.dart` (menu de pausa recebe toggle)

**`PauseOverlay` novo (já é `Positioned.fill` após D):**
```
Stack
├── if (!reduceEffects):
│     BackdropFilter(sigmaX: 12, sigmaY: 12)
│     └── Container(color: Color(0x4DFFF8E7))  ← branco-creme 30%
```

**Animação entrada:** `AnimatedOpacity` 0→1 em 250ms. Saída: 200ms.

**Casos de teste:**

### E.3 — Desfazer 1 e 3

**Arquivos modificados:**
- `lib/presentation/screens/game/game_screen.dart` — handlers `_onUndo1` / `_onUndo3`
- `lib/presentation/controllers/game_notifier.dart` — método `undo(int steps)`

**Fluxo:**
1. Tap em Desfazer N → `AlertDialog` "Usar N desfazer? Restam X após isso."
2. Confirma → `gameNotifier.undo(steps)` → `inventoryNotifier.consume(ItemType.undoN)`
3. Animação reversa 300ms

**`gameNotifier.undo(steps)`:** `undoStack[0]` = estado imediatamente anterior, `undoStack[2]` = 3 moves atrás. Restaura `undoStack[steps - 1]`, descarta os estados mais recentes que o ponto restaurado.

**Desabilitar:** `InventoryItemButton` de Desfazer 3 desabilitado se `undoStack.length < 3` (independente do contador do inventário).

**Casos de teste:**
- Desfazer 1: estado correto após 1 move
- Desfazer 3: estado correto após 3 moves
- Desfazer 3 desabilitado se undoStack.length < 3
- Cancelar dialog não consome item
- Contador decrementado após confirmar

### E.4 — Bomba 2 e 3 + `BombSelectionOverlay`

**Arquivos novos:**
- `lib/presentation/widgets/bomb_selection_overlay.dart`

**Arquivos modificados:**
- `lib/data/models/game_state.dart` — `bombMode: BombMode?` (enum: `bomb2`, `bomb3`)
- `lib/domain/game_engine/game_engine.dart` — `removeTiles(List<Position>)`
- `lib/presentation/controllers/game_notifier.dart` — `enterBombMode`, `selectBombTile`, `confirmBomb`, `cancelBomb`
- `lib/presentation/screens/game/game_screen.dart` — handler `_onBomb`

**`BombSelectionOverlay`** (`Positioned.fill`):
```
Stack
├── Container(color: preto 60%)             ← dim geral sobre tudo
├── Positioned (mesma posição do BoardWidget na Column)
│   └── Stack
│       ├── BoardWidget (sem dim — visível através do overlay)
│       └── SelectionLayer (GestureDetector por tile, pulsação nos selecionados)
└── Column
    ├── Text instrução (topo)
    ├── Spacer
    └── Row [Cancelar] [Explodir] (rodapé)
```

A posição do `BoardWidget` dentro do overlay é obtida via `GlobalKey` no board original (mesmo padrão do `_headerKey` já existente).

**Lógica de seleção:**
- Tap em tile vazio → `SnackBar` "Selecione um tile com animal"
- Tap em tile não-vazio não-selecionado:
  - Bomba 3: adiciona à seleção (máx 3)
  - Bomba 2: se seleção vazia → seleciona; se seleção tem 1 e é adjacente → adiciona; se não-adjacente → substitui o tile 1 pelo novo
- Tap em tile selecionado → desmarca
- "Explodir" habilitado quando seleção completa (2 ou 3)
- Confirmar → `gameEngine.removeTiles(positions)` → `inventoryNotifier.consume` → animação 500ms → `cancelBomb`
- Cancelar → `gameNotifier.cancelBomb()` → sem consumo

**Adjacência Bomba 2:** `(r2-r1).abs() + (c2-c1).abs() == 1`

**Casos de teste:**
- Bomba 2 aceita tiles adjacentes horizontais e verticais
- Bomba 2 rejeita diagonal (substitui tile 1)
- Bomba 2 rejeita tiles com distância > 1 (substitui tile 1)
- Bomba 3 aceita quaisquer 3 tiles não-vazios
- Tap em vazio → SnackBar, sem seleção
- Cancelar não consome item
- Confirmar remove tiles do tabuleiro e consome item
- Overlay fecha após confirmar
