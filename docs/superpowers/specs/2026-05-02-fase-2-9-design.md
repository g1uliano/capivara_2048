| `lib/app.dart` | Trocar rota inicial de `HomeScreen` para `SplashScreen`; remover `precacheImage` existente (migrado para `SplashScreen`) |

### Configuração `flutter_native_splash` em `pubspec.yaml`
```yaml
flutter_native_splash:
  color: "#FFFFFF"          # fundo branco (ajustar para cor do tema se necessário)
  image: assets/images/splash/splash_logo.png
  android: true
  ios: true
  web: false
```
Rodar após adicionar: `dart run flutter_native_splash:create`

### Spec do widget `SplashScreen`
```
initState:
  - Inicia timer de 1500ms
  - Dispara _init() em paralelo

_init():
  - precacheImage de todos os assets de animais (tile + host)
  - precacheImage de fundo (assets/images/fundo.png)
  - Inicializa Hive se necessário
  - Marca _initDone = true
  - Chama _tryNavigate()

timer callback:
  - Marca _timerDone = true
  - Chama _tryNavigate()

_tryNavigate():
  - Se _initDone && _timerDone: Navigator.pushReplacement(HomeScreen)

Animação de entrada (flutter_animate):
  - Logo: fadeIn(600ms) + scale(begin: 0.95, end: 1.0, 600ms)

Animação de saída (antes do pushReplacement):
  - fadeOut(400ms)
```

### Critérios de aceite
- Splash nativa visível antes do Flutter estar pronto
- `SplashScreen` Flutter exibida por no mínimo 1,5s mesmo em dispositivo rápido
- Navegação automática para `HomeScreen` ao término da inicialização
- Nenhum `precacheImage` duplicado entre `SplashScreen` e `app.dart`

### Testes
- Verificação manual: fechar e reabrir o app — splash aparece, desaparece, `HomeScreen` carregada
- Verificar em dispositivo lento (emulador sem aceleração): splash não some antes de 1,5s

## Sub-entrega B — `GameOverItemScreen`

### Objetivo
Overlay dedicado ao fluxo de salvamento quando o tabuleiro trava com itens disponíveis. Vida consumida apenas após confirmação de desistência.

### Arquitetura
Overlay no `Stack` da `GameScreen` — consistente com `BombSelectionOverlay`, `PauseOverlay` e `GameOverModal` existentes.

### Fluxo completo
```
1. GameEngine detecta: sem movimentos válidos → isGameOver = true
2. GameNotifier._onGameOver():
   a. _getAvailableSavingItems(inv) → lista de ItemType com count > 0
   b. Se lista vazia → fluxo antigo (GameOverModal / consumir vida)
   c. Se lista não vazia → isAwaitingGameOverResolution = true
3. GameScreen observa isAwaitingGameOverResolution == true
   → renderiza Positioned.fill(GameOverItemOverlay) no topo do Stack
   → AbsorbPointer envolve conteúdo abaixo (tabuleiro, header, inventário)
4. GameOverItemOverlay:
   - Backdrop: Colors.black.withOpacity(0.75)
   - Card central: fundo sólido AppColors.cardBackground,
     BorderRadius.circular(20), padding 24dp, largura 85% da tela
   - Exibe item atual em destaque (ícone 120×120dp, nome, mensagem, contagem)
   - Botões: "Usar item" | "Ver próximo" (se múltiplos tipos) | "Desistir"
   - Troca de item: fade-out/in 200ms
5. "Usar item" com undo:
   → inventoryProvider.consume(type)
   → gameProvider.notifier.undo(n)
   → isAwaitingGameOverResolution = false
   → overlay some, jogo continua
6. "Usar item" com bomba:
   → isAwaitingGameOverResolution permanece true
   → gameProvider.notifier.enterBombMode(mode, type)
   → BombSelectionOverlay abre (comportamento existente)
   → Cancelar bomba: isAwaitingGameOverResolution ainda true
     → GameOverItemOverlay reaparece com mesmo item
   → Confirmar célula: bomba executada
     → isAwaitingGameOverResolution = false
     → overlay some, jogo continua
7. "Desistir":
   → AlertDialog: "Tem certeza? Você perderá uma vida."
   → Cancelar: permanece na GameOverItemOverlay
   → Confirmar: livesNotifier.consumeLife()
     → isAwaitingGameOverResolution = false
     → Navigator.pushReplacement(HomeScreen) ou NoLivesScreen
```

### Ordem de ciclagem de itens (prioridade)
1. `undo1`
2. `undo3`
3. `bomb2`
4. `bomb3`

Apenas tipos com `count > 0` entram na lista. "Ver próximo" cicla circularmente.

### Arquivos modificados/criados
| Arquivo | Mudança |
|---------|---------|
| `lib/presentation/controllers/game_notifier.dart` | Adicionar `bool isAwaitingGameOverResolution = false`; método `_getAvailableSavingItems(Inventory)`; setar flag em `_onGameOver()`; setar `false` ao resolver |
| `lib/presentation/screens/game/game_over_item_overlay.dart` | **Novo.** Widget `GameOverItemOverlay` (StatefulWidget — gerencia item atual em exibição) |
| `lib/presentation/screens/game/game_screen.dart` | Adicionar `Positioned.fill` condicional para `GameOverItemOverlay`; `AbsorbPointer` no conteúdo abaixo quando overlay ativo |
| `lib/presentation/widgets/inventory_bar.dart` | Desabilitar `onPressed` de todos os itens quando `isAwaitingGameOverResolution == true` |

### Visual do card
```
┌─────────────────────────────┐
│   [ícone PNG 120×120dp]     │
│   Nome do item (20sp SB)    │
│   Mensagem explicativa      │
│   (16sp, max 2 linhas)      │
│   Você tem: N               │
│                             │
│  [Usar item]  [Ver próximo] │
│       [Desistir]            │
└─────────────────────────────┘
```
"Ver próximo" só aparece quando há ≥2 tipos de itens disponíveis.

### Critérios de aceite
- Tabuleiro trava com undo1 disponível → overlay exibe undo1 em destaque
- "Usar item" (undo) → tabuleiro desbloqueado, item consumido, overlay some
- "Usar item" (bomba) → `BombSelectionOverlay` abre; cancelar → overlay reaparece; confirmar célula → jogo continua
- "Desistir" → `AlertDialog`; cancelar → permanece no overlay; confirmar → vida consumida, navega para `HomeScreen`/`NoLivesScreen`
- Tabuleiro trava sem itens → overlay não exibido
- `InventoryBar` taps desabilitados durante overlay ativo
- Toque no backdrop fora do card: sem ação (não fecha o overlay)

### Testes unitários obrigatórios
- `_getAvailableSavingItems`: retorna lista vazia quando inventário zerado
- `_getAvailableSavingItems`: retorna apenas tipos com count > 0, na ordem correta
- `isAwaitingGameOverResolution` setado para `true` quando `isGameOver` + itens disponíveis
- `isAwaitingGameOverResolution` setado para `false` após undo executado
- `isAwaitingGameOverResolution` setado para `false` após desistência confirmada
