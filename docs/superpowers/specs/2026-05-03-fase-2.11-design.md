# Fase 2.11 — ShopOverlay acessível pelos ícones desabilitados do inventário

**Data:** 2026-05-03
**Status:** Aprovado — pronto para implementação
**Versão alvo:** v1.1.0

---

## Objetivo

Quando o jogador toca em um ícone de item com contador 0 (acinzentado) na `InventoryBar`, abrir a loja como overlay diretamente sobre o jogo — sem navegar para outra tela — permitindo comprar itens sem perder o contexto da partida. O cronômetro fica pausado enquanto o overlay estiver aberto. O pacote mais relevante para o item tocado fica destacado visualmente.

---

## Decisões de design

| Ponto | Decisão |
|---|---|
| Reutilização do card da loja | Extrair `_ShopPackageCard` para `ShopPackageCard` público em arquivo próprio |
| Feedback visual pós-compra | Pulso no ícone comprado (scale 1→1.2→1, 300ms) |
| Destaque de pacote relevante | Borda laranja `#FF8C42` nos pacotes que contêm o item tocado |
| Race condition pausa | Pausa compartilhada simples — `AbsorbPointer` impede coexistência física |

## Sub-A — Tap em ícone desabilitado

### Mudança de comportamento

- **Antes:** ícone com `count == 0` ignorava o tap (`onPressed: null`)
- **Agora:** ícone com `count == 0` chama `onTapWhenEmpty` com o `ItemType` correspondente

### `InventoryItemButton`

Novo parâmetro opcional:

```dart
final VoidCallback? onTapWhenEmpty;
```

Quando `count == 0`, o `GestureDetector` usa `onTapWhenEmpty` em vez de `null`. Visual acinzentado não muda.

Novo parâmetro opcional para pulso:

```dart
final bool shouldPulse; // default: false
```

Quando `shouldPulse` vira `true`, executa `animate().scale(begin:1, end:1.2).then().scale(end:1, duration:150ms)`.

### `InventoryBar`

Recebe dois novos parâmetros do `GameScreen`:

```dart
final void Function(ItemType) onTapWhenEmpty;
final Set<ItemType> pulsingItems;
```

Cada `InventoryItemButton` recebe:
- `onTapWhenEmpty: () => onTapWhenEmpty(itemType)` quando `count == 0`
- `shouldPulse: pulsingItems.contains(itemType)`

### Arquivos modificados
- `lib/presentation/widgets/inventory_item_button.dart`
- `lib/presentation/widgets/inventory_bar.dart`

## Sub-B — `ShopPackageCard` e `ShopOverlay`

### Extração do card

`_ShopPackageCard` extraído de `shop_screen.dart` para:

**`lib/presentation/widgets/shop_package_card.dart`**

```dart
class ShopPackageCard extends StatelessWidget {
  final ShopPackage package;
  final VoidCallback onBuy;
  final bool highlighted; // default: false
}
```

Quando `highlighted: true`, o `Card` recebe borda laranja `#FF8C42` de 2px. Aparência idêntica ao estado atual quando `highlighted: false`.

`ShopScreen` continua usando o card, agora importado, com `highlighted: false` sempre.

### Mapeamento de relevância

Nova função em `lib/data/shop_data.dart`:

```dart
List<String> packageIdsContaining(ItemType item)
```

Retorna os IDs dos pacotes que contêm ao menos 1 unidade do `item`. Lógica baseada nos dados já existentes em `kShopPackages`.

### `ShopOverlay`

**Arquivo novo:** `lib/presentation/widgets/shop_overlay.dart`

```dart
class ShopOverlay extends ConsumerStatefulWidget {
  final ItemType itemType;       // item que disparou a abertura
  final VoidCallback onClose;
  final void Function(ItemType) onItemPurchased;
}
```

**Layout:**

```
Stack (full-screen, PopScope canPop: true)
├── AbsorbPointer + ColoredBox(Colors.black.withOpacity(0.75))  ← fundo não interativo
└── SafeArea
    └── Column
        ├── Row (header)
        │   ├── OutlinedText("Loja", Fredoka 22, branco)
        │   └── IconButton(Icons.close, branco) → onClose()
        ├── Divider(branco, opacidade 0.3)
        └── Expanded → ListView.builder (padding 16dp)
            └── ShopPackageCard(highlighted: packageIdsContaining(itemType).contains(pkg.id))
```

**Fluxo de compra** (idêntico ao `ShopScreen`):
1. Tap "Comprar" → `AlertDialog` de confirmação
2. Confirmar → entrega itens ao `inventoryProvider` e `livesProvider`
3. `onItemPurchased(itemType)` chamado com o tipo principal do pacote comprado
4. Overlay permanece aberto — jogador fecha manualmente

**Botão voltar Android:** `PopScope(canPop: true)` → fecha via `onClose()`.

**Sem `AppBar` nativa** — header customizado dentro do overlay.

### Pulso pós-compra (coordenado pela `GameScreen`)

`ShopOverlay` chama `onItemPurchased(ItemType)` após cada compra confirmada. `GameScreen` implementa o handler: adiciona ao `_pulsingItems`, agenda `Timer(300ms)` para remover.

- **Novo:** `lib/presentation/widgets/shop_overlay.dart`
- **Modificado:** `lib/presentation/screens/shop_screen.dart` (usa `ShopPackageCard` importado)
- **Modificado:** `lib/data/shop_data.dart` (adiciona `packageIdsContaining`)

## Sub-C — Pausa do cronômetro e integração na `GameScreen`

### Estado local na `GameScreen`

```dart
ItemType? _shopItem;        // null = overlay fechado
Set<ItemType> _pulsingItems = {};
```

### Abrir overlay

```dart
void _openShop(ItemType type) {
  ref.read(gameProvider.notifier).pause();
  setState(() => _shopItem = type);
}
```

### Fechar overlay

```dart
void _closeShop() {
  setState(() => _shopItem = null);
  ref.read(gameProvider.notifier).resume();
}
```

### Pulso pós-compra

```dart
void _onItemPurchased(ItemType type) {
  setState(() => _pulsingItems.add(type));
  Timer(const Duration(milliseconds: 300), () {
    if (mounted) setState(() => _pulsingItems.remove(type));
  });
}
```

### Render no `Stack` da `GameScreen`

```dart
if (_shopItem != null)
  Positioned.fill(
    child: ShopOverlay(
      itemType: _shopItem!,
      onClose: _closeShop,
      onItemPurchased: _onItemPurchased,
    ),
  ),
```

### Guard de pausa

`GameNotifier.pause()` já tem `if (state.isGameOver || state.hasWon) return`.
`GameNotifier.resume()` já tem `if (!state.isPaused) return`.
`AbsorbPointer` interno da `ShopOverlay` impede fisicamente que `PauseButtonTile` seja tocado enquanto o overlay está aberto — sem risco de coexistência.

## Critérios de aceite

| Item | Critério |
|---|---|
| Tap | Ícone desabilitado (count == 0) abre `ShopOverlay` |
| Tap | Ícone habilitado (count > 0) continua abrindo `ConfirmUseDialog` |
| Overlay | Exibe 6 cards da loja |
| Overlay | Pacote(s) relevante(s) ao item tocado com borda laranja |
| Overlay | Conteúdo rolável verticalmente |
| Overlay | Botão X fecha e retoma cronômetro |
| Overlay | Botão voltar Android fecha e retoma cronômetro |
| Overlay | Tabuleiro visível ao fundo mas não interativo |
| Cronômetro | Para ao abrir, retoma exatamente do mesmo valor ao fechar |
| Compra | Entrega itens ao inventário (mesmo fluxo da ShopScreen) |
| Compra | Overlay permanece aberto após compra |
| Compra | Ícone do item comprado pulsa brevemente na InventoryBar |
| Regressão | ShopScreen comum sem alteração visual |
| Regressão | PauseOverlay continua funcionando normalmente |
