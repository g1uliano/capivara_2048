## Entrega B — `GameTitleImage`

### Decisões
- **Aleatorização:** por sessão (sortear no `initState` da `HomeScreen`, passar resultado como estado).
- **Animação:** estática nesta fase; animação `flutter_animate` reservada para Fase 2.6.

### Arquivo novo
`lib/presentation/widgets/game_title_image.dart`

### API
```dart
class GameTitleImage extends StatelessWidget {
  const GameTitleImage({super.key, this.height});
  final double? height;

  @visibleForTesting
  static String pickAsset({Random? random}) {
    final r = random ?? Random();
    return r.nextInt(2) == 0
        ? 'assets/images/title/title_orange.png'
        : 'assets/images/title/title_brown.png';
  }

  @override
  Widget build(BuildContext context) {
    // asset resolvido via pickAsset — chamado externamente (HomeScreen)
    // ou pode receber via construtor; ver integração abaixo
  }
}
```

**Alternativa de integração preferida:** `HomeScreen` sorteia o asset no `initState` e passa pra `GameTitleImage` via construtor `asset` — widget fica puramente passivo e mais fácil de testar em widget tests.

```dart
// HomeScreen (StatefulWidget)
late final String _titleAsset;

@override
void initState() {
  super.initState();
  _titleAsset = GameTitleImage.pickAsset();
}

// No build:
GameTitleImage(asset: _titleAsset, height: 220)
```

### API final do widget
```dart
class GameTitleImage extends StatelessWidget {
  const GameTitleImage({super.key, required this.asset, this.height});
  final String asset;
  final double? height;

  @visibleForTesting
  static String pickAsset({Random? random}) {
    final r = random ?? Random();
    return r.nextInt(2) == 0
        ? 'assets/images/title/title_orange.png'
        : 'assets/images/title/title_brown.png';
  }
}
```

### Integração na HomeScreen
- `HomeScreen` muda de `StatelessWidget` → `StatefulWidget`.
- `SizedBox(height: 220)` na linha 71 → `GameTitleImage(asset: _titleAsset, height: 220)`.

### pubspec.yaml — adicionar no bloco `assets:`
```yaml
- assets/images/title/
- assets/images/icon/
```

### Testes obrigatórios
| Teste | Tipo | Descrição |
|-------|------|-----------|
| `pickAsset` retorna orange | unit | `Random(0)` → `title_orange.png` |
| `pickAsset` retorna brown | unit | `Random(1)` → `title_brown.png` |
| Widget renderiza asset passado | widget | `GameTitleImage(asset: 'assets/images/title/title_orange.png')` renderiza `Image.asset` com path correto |
| HomeScreen usa asset válido | widget | `_titleAsset` é um dos dois paths válidos |

## Entrega C — Ícone do app

### Tight crop (passo obrigatório antes do gerador)

```bash
convert assets/images/icon/app_icon.png \
  -trim +repage \
  -gravity center \
  -background transparent \
  -extent 860x860 \
  assets/images/icon/app_icon_tight.png
```

Lógica: trim remove bordas transparentes; extent repadeia para 860×860 (~84% de 1024), deixando ~8% de margem em cada lado — dentro da safe zone Android adaptive icon (66% do centro).

### dev_dependency
```yaml
flutter_launcher_icons: ^0.14.0
```

### Configuração `flutter_launcher_icons:` no `pubspec.yaml`
```yaml
flutter_launcher_icons:
  android: true
  ios: true
  web:
    generate: true
    image_path: assets/images/icon/app_icon_tight.png
  windows:
    generate: true
    image_path: assets/images/icon/app_icon_tight.png
  image_path: assets/images/icon/app_icon_tight.png
  adaptive_icon_background: '#D4F1DE'
  adaptive_icon_foreground: assets/images/icon/app_icon_tight.png
```

### Geração
```bash
dart run flutter_launcher_icons
```

### Arquivos gerados — commitar
- `android/app/src/main/res/mipmap-*/ic_launcher*.png`
