## Sub-entrega 2 — Sistema de vidas

### Modelo de dados — `LivesState`

```dart
// Hive TypeAdapter gerado via hive_generator
class LivesState {
  int lives;                  // atual, 0–15
  int maxLives;               // 5 = regen cap padrão | 15 = inventory cap | -1 = ilimitado
  DateTime lastRegenAt;       // timestamp da última vida ganha por regen
  int adWatchedToday;         // contador diário de mock-anúncios
  DateTime adCounterResetAt;  // próxima meia-noite local (zera adWatchedToday)
  String? userId;             // null = local; preenchido na Fase 3
  DateTime? lastSyncedAt;     // null = nunca sincronizado
}
```

### Regras de negócio

- **Regen ao abrir o app:**
  `delta = now - lastRegenAt`
  Se `maxLives == -1`: `gained = floor(delta.inMinutes / 30)` (sem cap)
  Senão: `gained = min(floor(delta.inMinutes / 30), maxLives - lives)`
  `lives += gained`; `lastRegenAt += Duration(minutes: gained * 30)`
- **Consumo:** `-1` ao confirmar Novo Jogo; retomar jogo pausado não consome
- **Mock-anúncio:** `adWatchedToday++`; se `>= 40`, botão desabilitado; zera quando `now > adCounterResetAt`
- **Cap de exibição:** `LivesIndicator` exibe no máximo 5 corações; se `lives > 5`, mostra `❤️ ×N`

### Repositório — `LivesRepository`

- Usa Hive box `'lives'`
- Métodos: `load()`, `save(LivesState)`, `applyRegen()` (calcula delta e salva)

### Notifier — `LivesNotifier`

- `StateNotifier<LivesState>`
- Chama `applyRegen()` no construtor (ao inicializar o provider)
- Expõe: `consume()`, `rewardFromAd()`, `canWatchAd`

### Widget — `LivesIndicator`

- Fileira de corações preenchidos/vazios (máximo 5 visíveis)
- Número inline se `lives > 5`
- Tap → `OverlayEntry` flutuante com "próxima vida em MM:SS", auto-dismiss após 3s
- Presente na HomeScreen (topo) e no rodapé do GameScreen

### Widget — `NoLivesScreen`

- Modal fullscreen (rota empilhada sobre GameScreen ou HomeScreen)
- Exibe contagem regressiva até próxima vida
- Botão "Assistir anúncio": abre `Dialog` mock ("Anúncio simulado" + botão Fechar) → `LivesNotifier.rewardFromAd()` → `+1 vida` → dismiss → jogo retoma estado pausado
- Se chamada da HomeScreen (sem jogo ativo): dismiss retorna pra HomeScreen
- Botão desabilitado se `adWatchedToday >= 40`

### Critério de aceite

- Regen funciona com app fechado 30min+ (teste: manipular `lastRegenAt` diretamente)
- Overlay timer some em 3s
- Mid-game sem vidas → mock-anúncio → jogo retoma (não reinicia)
- `adWatchedToday` não ultrapassa 40; reseta após meia-noite

## Sub-entrega 3 — Anfitrião refatorado + StatusPanel + pause flutuante

### Atualização do modelo `Animal`

```dart
final String? hostSvgPath;           // null → fallback pro tile svg
final double? hostAspectRatio;       // null → 1.0
final String? backgroundTexturePath; // null → CustomPainter placeholder
```

### Widget — `HostArtwork`

- Renderiza `hostSvgPath` se não-nulo; fallback automático pro `tile/` svg correspondente
- Proporção via `hostAspectRatio` (default 1.0)
- Sem moldura de tile

### Widget — `HostBanner` (refatorado)

- Largura = 2 colunas + spacing (alinhado às colunas 1–2 do tabuleiro)
- Posição: canto superior esquerdo do GameScreen
- Contém `HostArtwork` + nome do animal abaixo

### Widget — `StatusPanel`

- Posição: canto superior direito, largura = 2 colunas + spacing (alinhado às colunas 3–4)
- Conteúdo (de cima pra baixo):
  - Cronômetro `HH:MM:SS` (fonte menor fixa, sem troca de formato)
  - Pontuação atual
  - Recorde
- Extrai lógica de cronômetro do `ScorePanel` existente

### Botão pause

- `Positioned` no canto superior direito da tela, fora do StatusPanel
- Sempre visível durante o jogo

### Critério de aceite

- Fallback automático pra tile svg quando `hostSvgPath == null`
- `HH:MM:SS` sem overflow na largura de 2 colunas
- Pause flutuante visível e funcional

## Sub-entrega 4 — Fundo dinâmico com textura

### Widget — `GameBackground`

Estrutura em Stack:
1. `Container` com cor `Color.lerp(animal.borderColor, AppColors.mint, 0.65)`
2. Camada de textura com opacidade 10–15%:
   - Se `animal.backgroundTexturePath != null` e asset carrega: renderiza imagem repetida
   - Se path nulo ou asset falha: `CustomPainter` com padrão geométrico por animal (pontilhado, hachura diagonal, grade, etc.) — cai silenciosamente, sem erro ao usuário
3. `RepaintBoundary` envolve o tabuleiro (evita rebuild durante crossfade)

### Mapeamento de texturas placeholder (CustomPainter)

| Grupo de animais | Padrão geométrico |
|---|---|
| Tanajura, Lobo-guará | Pontilhado fino |
| Sapo-cururu, Tucano | Hachura diagonal |
| Arara-azul, Preguiça | Grade leve |
| Mico-leão-dourado, Boto | Ondas horizontais |
| Onça-pintada | Manchas irregulares |
| Sucuri | Escamas (losangos) |
| Capivara Lendária | Radial (raios do centro) |

### Crossfade

- `AnimatedSwitcher` com duração 400ms ao trocar animal anfitrião
- Fundo não anima durante pause (overlay cobre tudo)

### Critério de aceite

- Tabuleiro não rebuilda durante troca de animal (verificar com Flutter DevTools)
- Fallback geométrico sem crash se asset inexistente ou com falha
- Crossfade visível ao formar novo animal anfitrião
