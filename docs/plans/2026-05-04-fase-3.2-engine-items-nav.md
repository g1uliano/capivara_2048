## Contexto técnico

### Boot patterns estabelecidos

```dart
// Para GameScreen
Future<void> _bootToGame(WidgetTester tester, GameTestHarness harness) async {
  final widget = await tester.runAsync(() => harness.boot());
  await tester.pumpWidget(widget!);
  await tester.pumpAndSettle(const Duration(seconds: 5));
  await tester.gotoGame(harness);
}

// Para HomeScreen
Future<void> _bootToHome(WidgetTester tester, GameTestHarness harness) async {
  final widget = await tester.runAsync(() => harness.boot());
  await tester.pumpWidget(widget!);
  await tester.pumpAndSettle(const Duration(seconds: 5));
}
```

### Pumping strategy (consolidada)
- Após `debugSetState` + state changes: `pump()` + `pump(Duration(milliseconds: 500))`
- Após gestos/taps em jogo: `pump()` + `pump(Duration(milliseconds: 500))` (NÃO pumpAndSettle — trava)
- `pumpAndSettle(Duration(seconds: 5))` só para splash/boot

### Game engine (game_notifier.dart)
- `harness.container.read(gameProvider.notifier).onSwipe(Direction.left)` — move direto, mais confiável que gesto
- `onSwipe` verifica `!isGameOver && !isPaused` antes de processar
- Após merge: `score += 1 << newLevel` onde `newLevel = oldLevel + 1`
- No-op swipe: `!anyChanged` → retorna estado sem spawn novo tile, sem mudar score
- GameOver: board cheio E sem tiles adjacentes do mesmo nível

### Inventory
- `onPressed: inventory.undo1 > 0 && undoStackLen >= 1 ? useUndo1 : null`
- `forceDisabled: inventory.undo1 > 0 && undoStackLen < 1` → botão visível mas desabilitado
- `BombDimOverlay` é renderizado quando `state.bombMode != null`
- `BombGridOverlay` aparece sobre o board quando `state.bombMode != null`

### Nav keys na HomeScreen
- `Key('home_btn_colecao')` → CollectionScreen ✅ (já feito)
- `Key('home_btn_configuracao')` → SettingsScreen ✅ (já feito)
- `Key('home_btn_recompensas')` → DailyRewardsScreen ✅ (já feito nos flow tests)
- `Key('home_btn_ranking')` → RankingScreen
- `Key('home_btn_loja')` → ShopScreen (tela, não overlay)
- `Key('home_btn_comojogar')` → `showModalBottomSheet(...)` (bottom sheet)

```dart
import 'package:capivara_2048/data/models/game_state.dart';
import 'package:capivara_2048/data/models/tile.dart';
import 'package:capivara_2048/domain/game_engine/direction.dart';
import 'package:capivara_2048/presentation/controllers/game_notifier.dart';
import '../_harness/test_harness.dart';
import '../_harness/tester_extensions.dart';

/// Cria board 4×4 vazio
List<List<Tile?>> _emptyBoard() =>
    List.generate(4, (_) => List<Tile?>.filled(4, null));

/// Seta um GameState direto no container (usa debugSetState que chama _stopTimer)
void _setState(GameTestHarness harness, GameState s) =>
    harness.container.read(gameProvider.notifier).debugSetState(s);

/// Lê o GameState atual
GameState _readState(GameTestHarness harness) =>
    harness.container.read(gameProvider);

/// Executa um swipe via notifier (confiável em testes headless)
void _swipe(GameTestHarness harness, Direction dir) =>
    harness.container.read(gameProvider.notifier).onSwipe(dir);
```

```dart
  tags: {ScenarioTag.critical},
  run: (tester, harness) async {
    await _bootToGame(tester, harness);

    // Primeiro merge: 2×level:1 → level:2, score += 2^2 = 4
    final board = _emptyBoard();
    board[0][0] = Tile(id: 'a', level: 1, row: 0, col: 0);
    board[0][1] = Tile(id: 'b', level: 1, row: 0, col: 1);
    _setState(harness, GameState(

      isGameOver: false, hasWon: false,

    await tester.pump();
    await tester.pump(const Duration(milliseconds: 500));

    final state1 = _readState(harness);
    expect(state1.score, equals(4)); // 2^2 = 4

    // Segundo merge: setup manual de mais 2 tiles level:1
    final board2 = state1.board.map((r) => List<Tile?>.from(r)).toList();
    // Colocar 2 level:1 em posições livres

    await tester.pump(const Duration(milliseconds: 500));

    final state2 = _readState(harness);
```
