# Fase 3.5 — Golden Tests com Alchemist

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Adicionar 15 golden tests (5 telas × 3 viewports) usando `alchemist` para snapshots visuais determinísticos do `HomeScreen`, `GameScreen`, `PauseOverlay`, `CollectionScreen` e `DailyRewardsScreen`.

**Architecture:** Os golden tests usam a API `goldenTest()` do alchemist (que cria seus próprios `testWidgets` internamente) e são registrados via `runGoldenTests()` chamado diretamente do `main()` de `run_all_test.dart` — fora do loop de `E2EScenario`. O harness é inicializado em `setUp/tearDown` padrão do Flutter test (antes de cada `testWidgets`). Cada tela usa `BoxConstraints.tight(Size(w, h))` para renderizar no tamanho exato do viewport. CI mode habilitado (`obscureText: true`) — texto vira blocos coloridos para consistência cross-platform. Platform goldens desabilitados.

**Tech Stack:** Dart / Flutter test · `alchemist ^0.14.0` (já no pubspec) · `GameTestHarness` existente · `UncontrolledProviderScope`

## Task 2: Criar `test/e2e/golden/golden_tests.dart`

Arquivo central com todos os 15 golden tests. Expõe a função `runGoldenTests()` que registra os grupos via `goldenTest()`.

**Files:**

- Create: `test/e2e/golden/golden_tests.dart`

**Conceito-chave:** `setUp(() async { ... })` do Flutter test roda ANTES do corpo do `testWidgets` que `goldenTest()` cria internamente. Isso garante que `_harness.container` já está populado quando `builder()` é avaliado.

**Como o alchemist captura o viewport:** ao passar `constraints: BoxConstraints.tight(Size(w, h))`, o alchemist aplica um `OverflowBox` com aquele tamanho e depois seta `tester.view.physicalSize = childSize`. O screenshot captura o widget renderizado naquele tamanho.

import 'package:flutter/material.dart';
import 'package:flutter_test/flutter_test.dart';

/// Envolve [child] no UncontrolledProviderScope do harness + MaterialApp com
/// o tema da app. Usado por todos os golden tests.
Widget _wrap(GameTestHarness harness, Widget child) {
  return UncontrolledProviderScope(
    container: harness.container,
    child: MaterialApp(
      debugShowCheckedModeBanner: false,
      theme: AppTheme.light(),
      home: child,
    ),
  );
}

/// Tabuleiro mid-game com tiles nos níveis 1–4 em posições espalhadas.
GameState _midGameState() {
  final board = List.generate(4, (_) => List<Tile?>.filled(4, null));
  board[0][0] = const Tile(id: 'g1', level: 3, row: 0, col: 0);
  board[0][2] = const Tile(id: 'g2', level: 1, row: 0, col: 2);
  board[1][1] = const Tile(id: 'g3', level: 4, row: 1, col: 1);
  board[1][3] = const Tile(id: 'g4', level: 2, row: 1, col: 3);
  board[2][0] = const Tile(id: 'g5', level: 1, row: 2, col: 0);
  board[2][2] = const Tile(id: 'g6', level: 2, row: 2, col: 2);
  board[3][1] = const Tile(id: 'g7', level: 3, row: 3, col: 1);
  board[3][3] = const Tile(id: 'g8', level: 1, row: 3, col: 3);
  return GameState(
    board: board,
    score: 420,
    highScore: 840,
    isGameOver: false,
    hasWon: false,
    maxLevel: 4,
  );
}

/// Registra todos os golden tests. Chamado pelo main() de run_all_test.dart.
void runGoldenTests() {
  _homeScreenGoldenTests();
  _gameScreenGoldenTests();
  _pauseOverlayGoldenTests();
  _collectionScreenGoldenTests();
  _dailyRewardsScreenGoldenTests();
}

// ---------------------------------------------------------------------------
// HomeScreen (3 viewports)
// ---------------------------------------------------------------------------

void _homeScreenGoldenTests() {
  group('golden — HomeScreen', () {
    for (final size in _viewports) {
      final w = size.width.toInt();
      final h = size.height.toInt();

      late GameTestHarness harness;

      setUp(() async {
        harness = GameTestHarness();
        await harness.boot();
      });

      tearDown(() async {
        await harness.teardown();
      });

      goldenTest(
        'home_${w}x${h}',
        fileName: 'home_${w}x${h}',
        constraints: BoxConstraints.tight(size),
        // pumpBeforeTest: onlyPumpAndSettle (default) — HomeScreen é estática.
        builder: () => _wrap(harness, const HomeScreen()),
      );
    }
  });
}

// GameScreen (3 viewports) — board mid-game pré-configurado

## Task 6: Troubleshooting — possíveis falhas e soluções

Esta task documenta problemas conhecidos e suas soluções. Não requer execução em condições normais — consulte se algum passo anterior falhar.

**Files:** nenhum (apenas documentação)

- [ ] **Problema: `PauseOverlay` não aparece no golden**

Sintoma: `pause_*.png` mostra o tabuleiro sem o overlay de pausa.

Causa provável: `find.bySemanticsLabel('Botão Pausar')` não encontrou o widget porque os semantics não estão ativos no ambiente de teste.

Solução alternativa: substituir o `pumpBeforeTest` por `find.byKey`:

```dart
pumpBeforeTest: (tester) async {
  // Alternativa: usar find.text se semantics não funcionar
  final pauseFinder = find.bySemanticsLabel('Botão Pausar').evaluate().isNotEmpty
      ? find.bySemanticsLabel('Botão Pausar')
      : find.text('Pausar');
  if (pauseFinder.evaluate().isNotEmpty) {
    await tester.tap(pauseFinder);
    await tester.pumpAndSettle();
  }
},
```

Se o texto também não funcionar, adicionar uma `Key` ao `PauseButtonTile` em `game_header.dart`:

```dart
// Em game_header.dart, passar key ao PauseButtonTile:
PauseButtonTile(
  key: const Key('pause_button'),
  tileSize: tileSize,
  onTap: onPause,
)
// E no golden test:
final pauseFinder = find.byKey(const Key('pause_button'));
```

- [ ] **Problema: golden file gerado é 1×1 pixel ou muito pequeno**

Sintoma: `ls -lh` mostra arquivo < 1KB.

Causa: `constraints` não foi aplicado corretamente. Verificar se `BoxConstraints.tight(size)` está sendo passado corretamente.

Solução: checar se o `goldenTest` está recebendo o parâmetro `constraints` (não `constraint` ou outro typo).

- [ ] **Problema: `setUp` async não funciona com alchemist**

Sintoma: `_harness.container` lança `LateInitializationError`.

Causa: em algumas versões do Flutter test, `setUp` com `Future` pode não funcionar com `goldenTest`.

Solução alternativa: usar `pumpBeforeTest` para botar o harness:

```dart
// Em vez de setUp(), inicializar em pumpBeforeTest.
// Passar o harness via late var e usar tester.runAsync():
late GameTestHarness _harness;

goldenTest(
  'home_360x640',
  fileName: 'home_360x640',
  constraints: BoxConstraints.tight(const Size(360, 640)),
  pumpBeforeTest: (tester) async {
    await tester.runAsync(() => _harness.boot());
    await tester.pumpAndSettle();
  },
  builder: () {
    _harness = GameTestHarness(); // late init no builder
    return _wrap(_harness, const HomeScreen()); // container ainda não inicializado aqui!
  },
);
```

**Atenção:** Esta alternativa tem problema pois `builder()` é chamado ANTES de `pumpBeforeTest`. Use a abordagem de `setUp` como primeira opção. Se não funcionar, refatorar para passar o widget via `pumpWidget` override (ver abaixo).

**Solução robusta via `pumpWidget` override:**

```dart
late GameTestHarness _harness;

goldenTest(
  'home_360x640',
  fileName: 'home_360x640',
  builder: () => const SizedBox.shrink(), // placeholder ignorado
  pumpWidget: (tester, _) async {
    _harness = GameTestHarness();
    addTearDown(_harness.teardown);
    await tester.runAsync(_harness.boot);

    tester.view.physicalSize = const Size(360, 640);
    tester.view.devicePixelRatio = 1.0;
    addTearDown(tester.view.resetPhysicalSize);
    addTearDown(tester.view.resetDevicePixelRatio);

    await tester.pumpWidget(_wrap(_harness, const HomeScreen()));
    await tester.pumpAndSettle();
    // NOTA: quando pumpWidget é overridden, alchemist não encontrará
    // o rootKey — precisamos usar matchesGoldenFile diretamente:
    await expectLater(
      find.byType(MaterialApp),
      matchesGoldenFile('goldens/ci/home_360x640.png'),
    );
  },
);
```

Esta última abordagem bypassa alchemist completamente para a captura, mas usa `alchemist` para o setup de CI mode via `flutter_test_config.dart`.
